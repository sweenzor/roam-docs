Delete Graph

markdown version · view in Roam Research

  • Deleting a graph is permanent. Once deleted, it cannot be recovered.
  • Make a backup first with Export, and use Export attachments in Data & Backups if you want copies of your uploaded files, because there is no undo.
  • Deleting a graph does not delete the files uploaded to it. If you want those gone too, delete them in the File Manager before deleting the graph.

How to delete a graph

  • Open Settings → Graph → Delete this graph. On shared graphs, only admins can do this.
  • Roam asks you to type the graph's name to confirm, then click I understand the consequences, proceed with deletion.

Deleting an encrypted graph

  • Deleting an [encrypted graph](Encrypted Graphs) also loses every file uploaded to it, even if you made a backup first, because backups don't include the media files.
  • Save the files first with Export attachments, in Settings → Data & Backups.

Related

  • Delete local cache, in the same settings tab, is a different thing: it clears the graph's copy on your device, and any unsynced local changes are lost with it. A hosted graph re-downloads from the server afterwards.
  • A Local Graph exists only in your browser, so deleting one removes it completely.
